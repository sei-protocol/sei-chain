# gensetup

Precompile version management tool, triggered via `go generate`.

## What it does

When a new upgrade version is added to `app/tags`, this tool:

1. Detects which precompile modules changed (via go-git)
2. Archives their current code into `precompiles/<module>/legacy/<folder>/`
3. Regenerates all `precompiles/<module>/setup.go` files from the `versions` files

It replaces the old `scripts/bump-version.sh` shell script.

## Developer workflow

To release a new upgrade version:

```bash
# 1. Make and commit your precompile code changes
git add precompiles/bank/bank.go
git commit -m "bank: add new functionality"

# 2. Add the upgrade name to app/tags
echo "v7.0" >> app/tags

# 3. Regenerate
go generate ./...

# 4. Commit the generated archives + setup.go changes
git add app/tags precompiles/
git commit -m "bump to v7.0"
```

## Naming conventions

| Concept | Format | Example |
|---------|--------|---------|
| Upgrade name (stored in `app/tags`, `versions`, setup.go map keys) | `vMajor.Minor` | `v7.0` |
| Legacy folder / package name | dots removed from `vMajor.Minor.0` | `v700` |

### Input sanitization

The tool only validates the **latest** entry in `app/tags` (new versions). Old 3-part entries like `v6.4.0` are left untouched.

| Input in `app/tags` | Behavior |
|---------------------|----------|
| `v7.0` | Used as-is |
| `v7.0.3` | Patch dropped with WARNING, `app/tags` rewritten to `v7.0` |
| `v6.4` | Error: semver-equal to existing `v6.4.0` |
| `foo`, `v6`, `v7.0-rc1` | Error: not a valid `vMajor.Minor` |
| Tag older than current latest | Error: "upgrade names must advance forward" |

## How change detection works

The tool compares **committed tree hashes**, not the working tree:

1. Searches commit history for the commit that introduced the previous version string in `app/tags` (pickaxe-style content search)
2. For each module directory, compares its tree hash between that commit and HEAD
3. Only modules whose tree hash changed are archived

> **Important:** Commit your precompile changes before running `go generate`. Uncommitted changes are not detected by the diff.

If `common/` changed, all modules are archived (they depend on it).

## Idempotency

Reruns are safe. The tool checks if `precompiles/common/legacy/<folder>/` exists — if yes, it skips archival and only regenerates `setup.go` files.

## Testing selective archiving

```bash
# Make a change to ONE module and commit it
echo "// test" >> precompiles/bank/bank.go
git add precompiles/bank/bank.go && git commit -m "test"

# Add a new version and generate
echo "v7.0" >> app/tags
go generate ./...

# Verify: only bank + common got archived
ls precompiles/bank/legacy/v700/    # exists
ls precompiles/oracle/legacy/v700/  # doesn't exist

grep "v7.0" precompiles/*/versions  # only bank/versions

# Rollback
git checkout -- app/tags precompiles/
git reset --soft HEAD~1
git checkout -- precompiles/bank/bank.go
```

## Safety fallbacks

All git failures fall back to archiving **all modules** — redundant but never incorrect:

| Failure | Fallback |
|---------|----------|
| Can't open repo / get HEAD | archive all |
| Commit search fails | archive all |
| Previous version not found in history | archive all |
| Tree comparison fails for a module | treat as changed |

## Files generated

- `precompiles/<module>/legacy/<folder>/*.go` — archived code with package/import rewrites
- `precompiles/<module>/legacy/<folder>/abi.json` — archived ABI
- `precompiles/common/legacy/<folder>/*.go` — archived common utilities
- `precompiles/<module>/setup.go` — regenerated from `versions` file
- `precompiles/<module>/versions` — appended with new version (changed modules only)
- `app/tags` — normalized if input had a patch

All generated `.go` files carry a `// Code generated by cmd/gensetup; DO NOT EDIT.` header.
